## 💻 Commodity Price Forecasting: WTI Crude Oil & Natural Rubber Cross-Market ML Pipeline

This repository contains the full, reproducible research pipeline behind the MSc thesis on cross-market price transmission between WTI crude oil and SHFE natural rubber. The codebase is organized as a modular, out-of-sample data pipeline — data ingestion, feature engineering, predictive modeling, event study, and scenario forecasting are each isolated into their own section for transparency and reproducibility.

### 1. Data Ingestion & Raw Inputs
* **`oil_data_10y.py` / `oil_price_10y.py` / `rubber_price.py` / `get_price.py`**: Scripts for extracting and validating a decade of historical WTI crude oil price/inventory data and SHFE natural rubber pricing data.
* **`data/oil_stocks_10y.csv`, `data/oil_prices_10y.csv`, `data/rubber_cleaned.csv`**: Raw cleaned inputs consumed directly by the notebook.

### 2. Core Analysis Notebook
* **`oil_prediction_comments_english.ipynb`**: The central, self-contained research notebook. Runs top to bottom and regenerates every processed dataset, model, and figure in this repo. Organized into labeled sections mirroring the thesis chapters:
  * **Section A–B** — data merging, inventory-deviation and geopolitical-event feature engineering.
  * **Section C** — ARIMA vs. Random Forest vs. XGBoost oil price forecasting (train 2014–2023, test 2024–2026, strict out-of-sample split). All models predict next-day **returns**, then reconstruct price via `today's actual price × (1 + predicted return)` to avoid data leakage.
  * **Section D / D2 / D3** — feature importance, and a true out-of-sample case study of the 2022 Russia–Ukraine shock (training cut off ~8 months before the invasion, so the model never sees the event during training).
  * **Section E–F** — oil–rubber cross-market data preparation (including the verified 2024/2025 Southern Thailand flood events) and Granger causality testing (oil returns → rubber returns).
  * **Section G** — cross-market rubber forecasting model (Random Forest vs. XGBoost), same return-then-reconstruct methodology, train 2015–2023 / test 2024–2026.
  * **Section H** — independent event study: because event dummies are zero-variance in the training window, ML feature importance for events is mechanically ≈0. This section separately tests event-period vs. non-event-period returns with Welch's t-test on the full sample, so the "no ML signal" and "no statistical significance" findings are not conflated.
  * **Section I** — dual-axis oil/rubber macro co-movement chart with all historical events annotated.
  * **Section J / J2** — 30-day-ahead scenario forecasting (Bull War / Base Flat / Bear Recession / Flood Shock), trained on the full 2015–2026 sample (a forward-looking tool, not an out-of-sample accuracy claim — kept separate from Section C/G's accuracy results).
  * **Section K** — transmission-lag sweep: rather than assuming a 5-day lag, this section builds true out-of-sample rubber models for lag N = 0–10 days and picks the lag with the lowest out-of-sample RMSE. **N = 5 days empirically minimizes RMSE** (226.1 vs. 232.1 at N = 0, a 2.6% improvement), giving the "5-day transmission lag" claim actual data support.

### 3. Processed Datasets (`data/`)
Auto-generated by running the notebook: `final_research_data.csv`, `final_research_data_enhanced.csv`, `master_oil_rubber_fixed.csv`, `lag_sweep_results.csv`.

### 4. Reference Scripts (`scripts/`)
Standalone, single-purpose versions of each notebook section (`section_A_data_prep.py` … `section_K_lag_sweep.py`) for readers who want to run or audit one step in isolation without the full notebook.

### 5. Visualization & Reporting (`figures/`)
Rendered chart outputs (feature importance, crisis-window RMSE comparison, Granger causality, macro co-movement, scenario forecasts, lag-sweep RMSE curve, etc.), each traceable to the notebook section that produced it.

### Key Findings
* **Three-tier early warning system** (for procurement decision-making): jointly monitors two signals with very different weights — **inventory deviation (55.70%)** and **oil momentum (14.43%)**. Inventory deviation dominates; oil momentum is a secondary confirming signal, not an equal partner.
* **Transmission lag**: empirically confirmed at 5 trading days (Section K), not assumed.
* **Event impact**: statistically tested independently of the ML models (Section H) — event-dummy feature importance being ≈0 in the ML models is a structural consequence of the chronological train/test split, and is unrelated to whether the events themselves moved prices.
